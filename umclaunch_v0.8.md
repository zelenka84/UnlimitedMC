# UnlimitedMC — техзадание проекта

Кастомный лаунчер Minecraft (только **Java Edition**) с модами, сборками, своим
оформлением и, в перспективе, аккаунтами и сообществом. Этот файл — единый бриф:
что за проект, как он должен выглядеть, что должен уметь и в каком порядке делать.

---

## 1. Что это и зачем

UnlimitedMC — свой лаунчер Minecraft с упором на удобную работу с модами и
сборками, аккуратный современный вид и поддержку игроков из РФ (где Modrinth и
часть сервисов заблокированы).

**Главные принципы (в порядке важности):**

1. **Сначала стабильность, потом дизайн.** Лаунчер не должен «отваливаться» после
   любого действия. Любая ошибка (нет сети, кривой ответ API, нет Java) показывается
   понятным сообщением — но **программа продолжает работать**, а не падает.
2. **Бесплатный, локальный.** Базовый лаунчер работает на компьютере пользователя и
   ничего не требует от сервера. Серверная часть (аккаунты, сообщество) — отдельный
   поздний этап.
3. **Только Java Edition.** Bedrock не делаем — там DRM, легально не выйдет.
4. **Заточен под РФ.** Встроенная поддержка прокси на весь трафик, потому что
   Modrinth (и его CDN) у нас блокируются.
5. **Честность.** Если функция не готова — она помечена как недоступная (заглушка),
   а не притворяется рабочей.

**Чего НЕ делаем:**

- Bedrock Edition (DRM).
- Пиратский обход покупки игры — оффлайн-режим только для запуска уже имеющейся игры
  и локального ника, не как способ «играть бесплатно на лицензионных серверах».

## 2. Текущее состояние (срез v0.8)

- **Python-версия — основная рабочая ветка** — `PySide6` + `minecraft-launcher-lib`
  + `requests`. Готовая библиотека делает тяжёлую работу по установке/запуску игры.
  Код разбит на модули: `unlimitedmc.py` (экраны + точка входа), `ui_kit.py`
  (дизайн-система: темы, акцент→градиент, логотип, ambient-фон, виджеты, анимации,
  векторные иконки, сборка QSS), `ui_content.py` (строки/новости/данные), `umc_core.py`
  (ядро без GUI: установка/запуск, Modrinth/CurseForge, миры/сервера/скриншоты, прокси,
  конфиг).
- **Дизайн-макет (HTML/CSS/JS, версия v6)** — кликабельный прототип внешнего вида.
  Это **не лаунчер**, а эталон. Реальный лаунчер уже подтянут к нему вплотную.

**Что уже умеет код (Python, v0.8):**

- **Оформление:** тёмная/светлая темы (тумблер), акцент из 5 свотчей → монохромный
  градиент, смена **вживую**; логотип-знак (два бара под 18°); реактивный ambient-фон
  с «дыханием» и плавным переливом под акцент/источник; полная локализация **RU/EN**.
  Все иконки интерфейса — **векторные SVG** (чёткие на любом шрифте, перекрашиваются
  под тему; эмодзи с кнопок убраны — на части систем они не отображались).
- **Окно:** **кастомная рамка под тему** (свой заголовок с брендом и кнопками
  свернуть/развернуть/закрыть, перетаскивание, изменение размера, лёгкое скругление
  углов). **Стилизованные окна ошибок/подтверждений** вместо системных.
- **Анимации:** скользящий тумблер, плавная смена разделов, каскадное появление
  карточек, подъём карточек и свечение кнопок при наведении, мягкие появления окон,
  плавная прокрутка к разделам.
- **Сайдбар:** сворачивается в иконки и раскрывается по наведению; активный пункт
  подсвечен акцентом.
- **Главная:** случайное приветствие, новости-карточки → **страница статьи**, баннер
  версии.
- **Играть:** локальный ник; сборки (создание / запуск / удаление). Превью сборки —
  **коллаж из 3–5 логотипов её модов** (скруглённые плитки). **Лаунчер сворачивается
  на время игры и возвращается после её закрытия.**
- **Менеджер сборки** (одно окно на сборку, см. 4.10): **Настройки** (переименование,
  RAM на сборку, открыть папку, удалить), **Контент** (моды / ресурспаки / шейдеры
  по файлам на диске — с разбивкой по типам, чипами-переходами и **настоящими
  логотипами**; удаление любого файла, включая добавленный вручную), **Миры**
  (импорт / экспорт `.zip`, удаление), **Сервера** (чтение `servers.dat`),
  **Скриншоты** (галерея, открыть, удалить).
- **Modrinth / CurseForge:** поиск модов / ресурспаков / шейдеров с фильтром по типу
  и учётом загрузчика/версии; **самые популярные грузятся сразу** (Modrinth — всегда;
  CurseForge — после ввода своего API-ключа); **постраничная навигация** (1 2 3 …);
  селектор «Ставить в:»; установка с **авто-зависимостями**; если мод уже установлен —
  вместо «Установить» красная «Удалить» с подтверждением. CurseForge — со встроенным
  ключом и **понятной ошибкой**, если ключ/квота не сработали. (Загрузка сборок-modpack
  убрана — только моды/паки/шейдеры.)
- **Логотипы модов:** сохраняются при установке; для добавленных вручную — опознаются
  по **SHA1 через Modrinth** (один запрос на сборку) и показываются в коллаже и в списке.
- **Настройки:** акцент, тема, язык, реактивный фон, RAM, путь к Java, прокси, ключ
  CurseForge, смена локального ника, повтор онбординга.
- **Стабильность:** тяжёлые операции в фоне; исключения не роняют лаунчер, а
  показываются понятным сообщением.

**Заглушки (ждут серверный этап):** аккаунты UnlimitedMC (вход/регистрация),
страница профиля, содержимое раздела «Сообщество» и блок «Друзья сейчас играют».
Везде, где нужен вход, стоит честная заглушка. Локальный игровой ник (офлайн) с
аккаунтом не связан.

**Главная задача дальше:** довести стабильность ядра (Forge/загрузчики, natives на
старых версиях) и затем подключить онлайн-часть (раздел 6).

## 3. Дизайн-система

Это раздел про «весь дизайн» — конкретные цвета, размеры и поведение. Эталон —
макет v6.

### 3.1. Темы

- **Тёмная** (по умолчанию) и **светлая**, переключаются тумблером.
- Тёмная палитра: фон `#090A0F`, сайдбар `#0c0e15`, карточки чуть светлее фона,
  основной текст `#E7EAF2`, приглушённый текст `~#8A92A6`, тонкие разделители
  `rgba(255,255,255,0.06)`.

### 3.2. Акцентный цвет

- Один акцентный цвет на всё приложение, выбирается из **5 свотчей**:
  - `#4C8DFF` — синий (по умолчанию)
  - `#7C5CFF` — фиолетовый
  - `#2FD06E` — зелёный
  - `#FB8C3C` — оранжевый
  - `#FF5C7A` — розовый
- Из выбранного акцента строится **монохромный градиент** (светлее→темнее того же
  цвета) для кнопок, активных элементов, прогресс-баров. В макете это сделано через
  CSS `color-mix` и `@property --accent`.
- Смена акцента применяется **вживую**, без перезапуска.

### 3.3. Логотип

- Знак — **две скруглённые «капсулы» (stadium-бары)**, наклонённые на **18°**,
  с непрерывным **диагональным градиентом** через обе (как один разрез цвета).
  Низы баров **выровнены** по одной линии.
- Цветной градиент логотипа: `#5C9CFF → #2461E8`. Есть серый (greyscale) вариант.
- Текст бренда: «Unlimited» обычным цветом + «**MC**» акцентным/синим.
- В макете знак — инлайн-SVG (два `rect`, повёрнутые `rotate(18 …)`, со скруглением
  `rx`, и общий линейный градиент по диагонали).

### 3.4. Фон

- **Реактивный ambient-фон**: мягкие размытые пятна акцентного цвета, чуть «дышат».
  Ненавязчивый, не мешает читать. Меняет оттенок вместе с акцентом.

### 3.5. Навигация / сайдбар

- Слева вертикальный **сайдбар**: бренд сверху, пункты меню, аккаунт снизу.
- Пункты: **Главная, Играть, Сообщество, Modrinth, CurseForge, Настройки**.
- На десктопе сайдбар **сворачивается в иконки и раскрывается по наведению**
  (hover). На мобильном — выезжающая шторка (drawer).
- Активный пункт подсвечен акцентом.

### 3.6. Локализация

- Полные **RU и EN** версии всего интерфейса (тексты, плейсхолдеры).
- В макете — словарь переводов + атрибуты `data-en` / `data-ph-en`, переключатель
  языка. В реальном лаунчере сделать аналогично (словарь строк).

### 3.7. Прочее

- Версия макета/сборки видна прямо в интерфейсе (на «Главной» в баннере и в
  заголовке окна) — удобно понимать, какая это итерация.
- Тёплые, человеческие микротексты (случайное приветствие на Главной и т.п.).

### 3.8. Рамка окна и анимации (реализовано в v0.5)

- **Кастомная безрамочная рамка** под тему: свой заголовок с брендом слева и кнопками
  свернуть / развернуть / закрыть справа (векторные иконки, перекрашиваются на ховере),
  перетаскивание за полосу, изменение размера за углы, лёгкое скругление углов окна.
- **Стилизованные окна-сообщения** (ошибки / инфо / подтверждение) в духе карточек
  интерфейса — вместо системных диалогов ОС.
- **Анимации (Qt не умеет CSS-transition, всё на `QPropertyAnimation`/таймерах):**
  скользящий тумблер с лёгкой пружинкой; плавная смена разделов; каскадное появление
  карточек; подъём карточек и свечение-нажатие кнопок при наведении; мягкое появление
  окон и диалогов; ambient-фон с переливом цвета при смене акцента/темы.
- **Производительность:** QSS пересобирается только при реальной смене темы/акцента
  (а не на каждой навигации), ambient рисует пятна по их площади, иконки кэшируются —
  переключение тем и переходы идут плавно.

## 4. Экраны (что на каждом)

### 4.1. Онбординг (первый запуск)

Мастер: приветствие → ввод ника в игре → выбор акцентного цвета → «Начать».
После него открывается основное окно. Повторно не показывается (флаг `onboarded`).

### 4.2. Главная

- Большой заголовок бренда, краткое описание.
- **Случайное приветствие** при каждом заходе.
- **Новости** — карточки, **кликабельные**: открывают полноценную **страницу статьи**.
- Баннер с номером версии (на месте бывшего рекламного блока — рекламы нет).
- Быстрые шаги «как начать» / кнопка перехода к сборкам.

### 4.3. Играть

- Выбор/отображение аккаунта (ник).
- Список **сборок (instances)** карточками: превью — **коллаж логотипов модов**,
  название, строка «загрузчик · версия · N модов» (моды считаются **по файлам в папке
  `mods`**). Кнопки на карточке: **▶ Играть** и **Менеджер сборки** (см. 4.10).
- Создание новой сборки (название + версия игры + загрузчик).
- Блок «**Друзья сейчас играют**» (онлайн-фича, появляется с сервером).

### 4.4. Modrinth

- Поиск модов / ресурспаков / шейдеров (сборки-modpack не скачиваем).
- **Самые популярные моды грузятся сразу** при заходе на страницу.
- **Фильтр по типу** (чипы), учёт загрузчика и версии выбранной сборки.
- **Постраничная навигация** «1 2 3 …» под результатами.
- Селектор «**Ставить в:**» — в какую сборку устанавливать.
- Карточки результатов: «Установить», а если мод уже в сборке — красная «Удалить»
  с подтверждением.
- Установка **обязательных зависимостей автоматически** (рекурсивно, без дублей).
- Подсказка про прокси: Modrinth в РФ заблокирован, при проблемах — вписать прокси.

### 4.5. CurseForge

- То же, что Modrinth, но через CurseForge API. **Популярные моды появляются после
  ввода своего API-ключа** в Настройках (бережём квоту встроенного ключа); ручной поиск
  работает и со встроенным ключом.
- Есть **встроенный ключ** (работает «из коробки»); свой бесплатный ключ с
  `console.curseforge.com` можно вписать в Настройки → CurseForge ради личной квоты.
- Если ключ отклонён или кончилась квота — **понятная ошибка** со ссылкой, а не сырой
  `403` (лаунчер продолжает работать). Встроенный ключ может быть отозван — это барьер
  от случайных утечек, а не криптозащита (надёжно прятать ключ можно только на релее).
- Корректно обрабатывать моды, запретившие скачивание вне CurseForge (понятная ошибка).

### 4.6. Сообщество (онлайн, нужен сервер)

- Вкладки: **Поиск** (поиск игроков) и **Друзья** (заявки, статусы
  online / dnd / invisible).
- **Гейтинг по входу**: без аккаунта показывается экран
  «Для того чтобы просматривать сообщество войдите в аккаунт UnlimitedMC.» + кнопка
  входа. С аккаунтом — реальное содержимое.

### 4.7. Профиль

- Профиль — это **полноценная страница**, а **не карточка** (важно):
  - Шапка **во всю ширину** в самом верху (на месте заголовка раздела).
  - **Аватар** (фото или инициалы), перекрывающий шапку.
  - Кнопка «Назад» — внизу, во всю ширину.
- У **чужого** профиля показывается активность (если играет и статус позволяет):
  мир, время в игре, сборка/версия. При статусе «не беспокоить» — только время.
- **Редактирование профиля — тоже отдельная страница** (не модалка): имя, «о себе»,
  аватар. Доступно владельцу и из настроек. Выход из аккаунта — на своей странице.

### 4.8. Настройки

- Выбор акцентного цвета (свотчи, применяется вживую).
- Отображаемое имя, «о себе».
- **Память (RAM)** — `-Xmx` / `-Xms`.
- **Путь к Java** (пусто = системная), кнопка «Обзор».
- **Прокси** — `http://host:port` или `socks5://host:port`, применяется ко **всему**
  трафику, включая загрузку игры.
- **Ключ CurseForge**.

### 4.9. Регистрация / вход

- В макете — упрощённо, в памяти (сбрасывается при перезагрузке). Реально — через
  сервер (см. раздел 6).
- Только **логин + пароль** (без email / Microsoft / Ely.by на этом этапе).
- Валидация: логин ≥3 символов, пароль ≥4, латиница/цифры/`_`, проверка занятости
  ника и неверного пароля.
- **Важно:** локальный игровой ник (для запуска оффлайн) — это **отдельная** сущность
  от аккаунта UnlimitedMC.

### 4.10. Менеджер сборки

Одно окно на сборку (открывается кнопкой на карточке в «Играть»). Слева — список
разделов с векторными иконками, справа — содержимое:

- **Настройки:** переименование, загрузчик и версия (только показ), **RAM на эту
  сборку** (`0` = брать общую из Настроек), «Открыть папку сборки», «Удалить сборку»
  (с подтверждением).
- **Контент:** моды / ресурспаки / шейдеры — **по реальным файлам на диске** (видно и
  добавленные вручную), с разбивкой на разделы, **чипами-переходами** к каждому разделу
  и **настоящими логотипами** (для ручных файлов логотип ищется по SHA1 через Modrinth).
  Удалить можно любой файл — с подтверждением, файл стирается с диска.
- **Миры:** список из `saves/`; **импорт** мира из `.zip`, **экспорт** мира в `.zip`,
  удаление.
- **Сервера:** сохранённые сервера из `servers.dat` (имя + адрес) — только просмотр.
- **Скриншоты:** галерея из `screenshots/`; клик по миниатюре — открыть в системе,
  корзина — удалить.

## 5. Функционал лаунчера (ядро)

- **Сборки (instances) в духе Prism Launcher:** каждая изолирована, своя папка
  `gameDirectory`, свои моды/настройки; превью сборки — **коллаж логотипов её модов**.
  Файлы игры (версии, библиотеки, ассеты) общие, чтобы не дублировать гигабайты.
- **Загрузчики:** vanilla, **Fabric**, **Forge**, **Quilt**, **NeoForge** — все через
  `minecraft-launcher-lib`. NeoForge ставится через модуль `mod_loader` (берётся
  последняя стабильная версия загрузчика под выбранную версию игры; доступен с 1.20.2).
- **Установка модов** с Modrinth и CurseForge, авто-зависимости, проверка «уже
  установлено». **Подсчёт и удаление контента — по реальным файлам в папках сборки**
  (`mods` / `resourcepacks` / `shaderpacks`), включая добавленные вручную; их логотипы
  опознаются по **SHA1 через Modrinth**.
- **Менеджер сборки:** настройки на сборку (RAM), контент (моды/паки/шейдеры), миры
  (импорт/экспорт `.zip`), сервера (чтение `servers.dat`), скриншоты.
- **Запуск оффлайн:** офлайн-UUID из ника, без онлайн-аутентификации Microsoft (её
  можно добавить позже как отдельную фичу).
- **Прокси** на весь сетевой трафик (Modrinth, CurseForge и загрузка игры) — критично
  для РФ.
- **Стабильность:** тяжёлые операции (установка, запуск) идут в фоновом потоке; любые
  исключения ловятся и показываются как «Не получилось …», лаунчер не падает, статус
  возвращается в рабочее состояние.
- **Параметры запуска:** RAM (`-Xmx`/`-Xms`) — глобально или **на отдельную сборку**,
  путь к Java (или системная).

### Где хранятся данные

Папка данных — по ОС:

- Windows: `%APPDATA%\UnlimitedMC`
- macOS: `~/Library/Application Support/UnlimitedMC`
- Linux: `$XDG_DATA_HOME/UnlimitedMC` или `~/.local/share/UnlimitedMC`

Внутри: `minecraft/` (общие версии, библиотеки, ассеты), `instances/<id>/` (папки
сборок), `config.json` (настройки и список сборок).

## 6. Аккаунты и серверы (онлайн-часть, поздний этап)

Базовый лаунчер работает без сервера. Аккаунты и сообщество — отдельный этап, его
делаем, когда сам лаунчер стабилен.

### 6.1. Две площадки

- **`umclaunch.net` — основной домен (международный).** Сервер в Нидерландах
  (или Германии). Работает как **релей**: проксирует Modrinth/Mojang, **ничего не
  хранит**. Чистый IP, который из РФ доступен напрямую (VPN нужен только чтобы пробить
  сам Modrinth). Раз ничего не хранит — регистрация бизнеса не требуется.
- **`unlimitedmc.ru` — запасной + хранилище.** Сервер в РФ. Здесь **база аккаунтов**,
  профили, сообщество. Хранение данных россиян в РФ удовлетворяет **152-ФЗ**.

### 6.2. Безопасность аккаунтов

- Пароли **хешируются** (bcrypt / argon2 + соль), **не шифруются** (хеш — в одну
  сторону). Свою криптографию не изобретать.
- Весь трафик — **HTTPS/TLS** (сертификат Let's Encrypt).
- При входе сервер выдаёт **токен с истечением** (он шлётся вместо пароля при
  дальнейших запросах).
- Хардненинг сервера: фаервол, вход по SSH-ключу, БД не торчит наружу, бэкапы.

### 6.3. Юридическое

- Бесплатный локальный лаунчер сам по себе регистрации не требует.
- Регистрация (**ИП / самозанятость**) включается, когда есть **доход** или
  **хранение персональных данных**. Релей (ничего не хранит) → без регистрации;
  аккаунты (хранят ПД) → нужен ИП. Оформление — через папу (помогает с ИП и
  хранением данных).

### 6.4. VPS

- Дешёвые VPS с оплатой из РФ: ~200–400 ₽/мес за штуку. Брать KVM + NVMe, оплата
  МИР/СБП, использовать триал.
- Локации: Германия / Нидерланды. Проверять правила провайдера по прокси/VPN и
  репутацию IP.
- Не покупать, пока лаунчер не работает. Можно стартовать **с одного** сервера.

## 7. Технологии

- **Основная ветка — Python:** `PySide6` (интерфейс Qt), `minecraft-launcher-lib`
  (установка/запуск игры), `requests` (сеть). Требует Python 3.10+ и Java 17+.
- **Альтернатива — C++:** `Qt6` + `CMake`. Без готовой библиотеки, поэтому больше
  кода и дыр. Держим для сравнения; решение, какую ветку тянуть дальше, — за
  владельцем после теста обеих.
- Выбор языка не влияет на «известность» лаунчера: Prism/MultiMC — C++/Qt,
  ATLauncher — Java, GDLauncher — Electron, Modrinth App — Rust/Tauri. Задача —
  **доделать**, а не переписать на «модный» язык.
- **Лицензия:** GNU **GPL-3.0** (файл `LICENSE` в корне; GitHub распознаёт её
  автоматически).

## 9. Короткий чек-лист «что важно не потерять»

- Стабильность важнее красоты, но красоту тоже доводим до макета.
- Один акцентный цвет → монохромный градиент; 5 свотчей; смена вживую.
- Логотип: два бара под 18°, диагональный градиент, низы выровнены.
- Профиль — **страница во всю ширину**, не карточка; редактирование — тоже страница.
- Сообщество закрыто без входа (точный текст-заглушка указан в 4.6).
- Прокси — на весь трафик (для РФ).
- Локальный игровой ник ≠ аккаунт UnlimitedMC.
- Каталог — только моды/паки/шейдеры; сборки-modpack не скачиваем.
- Контент и счётчик модов сборки — по реальным файлам на диске (видны и добавленные вручную).
- Иконки интерфейса — векторные SVG, не эмодзи.
- Bedrock не делаем. 
- `umclaunch.net` — релей без хранения; `unlimitedmc.ru` — хранилище под 152-ФЗ.
